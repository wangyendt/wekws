# ASR 筛洗与人工复核并行方案

## 当前状态

当前 [`data_xlxl_0327_ctc_v1`](/home/wayne/work/code/project/ffalcon/wekws_baseline/examples/hi_xiaowen/s0/data_xlxl_0327_ctc_v1) 不是“全量 ASR 转写完成版”。

准确说：

- 新 0327 正样本：当前是规则映射成 `小 雷 小 雷 / 小 雷 快 拍`
- 新 0327 `FILLER`：当前仍主要是 `<filler>`
- 旧 hi/ni 负样本：因为直接复用了旧 `data.list`，这些条目已经带原来的转写文本

所以它已经够支撑第一版 baseline，但还不是最终干净版。

## 建议顺序

1. 先对 `dev/test` 做 ASR 筛洗，优先把评测集弄干净。
2. 同时导出 `dev/test` 到本地，做人工听标/复核。
3. 只要 `dev/test` 这一层干净，baseline 就可以先起。
4. `train` 的进一步清洗可以在训练并行继续做，不必阻塞 baseline v1。

## 为什么不先把 train 清到极致

- 训练集允许少量噪声。
- 评测集不允许脏，否则 FAR/FRR 会失真。
- 现在最值钱的是尽快拿到一版 teacher baseline，而不是先把全部训练集整理到极致。

## 筛洗规则

脚本：[`screen_xiaolei_asr.py`](/home/wayne/work/code/project/ffalcon/wekws_baseline/examples/hi_xiaowen/s0/tools/screen_xiaolei_asr.py)

当前规则基于：

- ASR 文本精确命中
- ASR 文本包含目标词但带额外上下文
- ASR 文本命中另一目标词
- ASR 文本仅出现 `小雷 / 快拍` 等局部线索
- 完全未命中

输出分桶：

- `clean`
- `ambiguous`
- `reject`

说明：当前本地这版 `funasr==1.3.1` 默认结果稳定包含 `text` 和 `timestamp`，但没有一个现成稳定的逐词 confidence 字段可直接依赖，所以第一轮筛洗主要靠文本匹配和时间戳。

## 人工复核导出

脚本：[`export_xiaolei_review_package.py`](/home/wayne/work/code/project/ffalcon/wekws_baseline/examples/hi_xiaowen/s0/tools/export_xiaolei_review_package.py)

它会：

- 从 `dev/test` 的 `data.list` 读取样本
- 拷贝对应 wav 到一个 review package
- 输出 `manifest.jsonl`
- 输出 `manifest.csv`

之后再把 review package 打包下载到本地即可。

## 下载到本地

建议流程：

1. 在服务器上导出 review package 到 `/tmp/xlxl_review_package_dev_test`
2. 打包成 `/tmp/xlxl_review_package_dev_test.tar.gz`
3. 在本地电脑执行 `scp` 或 `rsync` 拉回

`scp` 模板：

```bash
scp your_user@your_server:/tmp/xlxl_review_package_dev_test.tar.gz ~/Downloads/
```

`rsync` 模板：

```bash
rsync -avP your_user@your_server:/tmp/xlxl_review_package_dev_test.tar.gz ~/Downloads/
```

## 并行建议

- 机器并行：先跑 `dev/test` 的 ASR 筛洗
- 人工并行：本地复核 `dev/test` 的 review package
- 训练并行：等 `dev/test` 基本可信后，直接起 baseline
- 后处理并行：再继续筛 `train`，准备 baseline v2
