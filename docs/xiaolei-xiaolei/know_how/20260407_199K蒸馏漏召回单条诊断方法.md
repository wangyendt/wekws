# 199K 蒸馏漏召回单条诊断方法

## 适用场景

当出现下面这种现象时，优先用本文的方法做单条语音诊断，而不是先继续扫阈值：

- 样本主观听感上是明显的 `小雷小雷`
- 199K 学生在 `test split` 上漏召回
- 392K 老师能够召回同类样本，或总体指标明显更好
- `analyze_exp_test_stats.py` 选出的工作点已经把阈值压到 `0.0`，但学生仍然 `rejected`

## 核心判断原则

先分清三类问题：

1. 阈值问题
2. 学生表征问题
3. 数据域偏移问题

最容易误判的是第 1 类。对当前小雷 199K 来说，很多样本学生侧 `threshold=0.0` 依然未触发，这时继续调阈值通常没有意义。

## 推荐流程

### 1. 先做集合级交叉

把学生漏掉的正样本和老师的 `score.txt` 做交叉，先看是：

- 老师也漏
- 还是老师能收、学生收不住

当前 `replay_from79/test_229` 的 `小雷小雷` 漏召回里：

- 学生漏召回 `47` 条
- 392K 老师能收回 `39` 条
- 只有 `8` 条连老师也漏

这说明当前主问题不是“样本都太难”，而是大部分普通样本其实老师能收、学生没学到。

### 2. 再跑同一条 wav 的 teacher / student 对照诊断

推荐直接用 [`diagnose_wav.py`](/home/wayne/work/code/project/ffalcon/wekws_baseline/examples/hi_xiaowen/s0/diagnose_wav.py)。

当前案例的固定命令已收录到命令表，核心有三条：

- `119.pt` 学生诊断
- `replay_from79/229.pt` 学生诊断
- `392K 159.pt` 老师诊断

诊断结果重点看四组字段：

- `infer_result.triggered / candidate_score / threshold`
- `greedy_decode.text` 与 `beam_topk[0]`
- `keyword_beam_topk` 里是否存在完整 `小雷小雷` 路径
- `mean_frame_topk_tokens` 与 `token_traces` 中 `小 / 雷 / blank` 的峰值和时序位置

## 当前小雷案例的判读模板

代表样本：`XIAOLEI_XIAOLEI_0031_1m_f_04_fast_01_rep_10`

### 现象

- `119.pt`：`rejected`
- `replay_from79/229.pt`：`rejected`
- `teacher 159.pt`：`detected 小雷小雷 0.999`

### 判读

- 两个 199K 学生的 `threshold_map[小雷小雷]` 都是 `0.0`，所以不是阈值问题。
- 两个 199K 学生的 `greedy_decode` 都塌成 `雷 小 雷`，说明完整 `小-雷-小-雷` 路径没有保住。
- 392K 老师的 `greedy_decode` 是 `<filler> 小 雷 小 雷`，并且 keyword beam 能形成稳定完整路径，因此可以高分触发。
- 从平均后验看，学生更偏向 `blank`，而 `小` 的非 blank 支撑明显弱于老师。

### 结论

当前 199K 学生的主要矛盾不是 feature 完全没学到，而是：

- 非 blank 激活过弱
- 第一个 `小雷` 易被 blank 吃掉
- 关键词内部时序结构没有被稳定保住

## 如何区分“学生表征不足”和“域偏移”

当前 47 条漏召回可分两类：

- 普通录音类：`20250205_rep / fast_rep / low_rep / other` 共 `32` 条，老师全部能收回
- 域偏移类：`hot_word_dsp / shangxu_other` 共 `15` 条，其中 `8` 条连老师也漏

因此后续实验要分开判断：

- 想补回那 `39` 条老师能收的样本，优先改学生训练策略
- 想补那 `8` 条老师也掉的样本，单靠蒸馏不一定够，更像要补 DSP / 上叙域数据或单独做域适配

## 对训练调整的直接启发

基于当前证据，下一步优先级应是：

1. 不要再把主要精力放在扫阈值上。
2. 先解决学生 `blank` 过强、非 blank 过弱的问题。
3. 先解决关键词内部时序路径保不住的问题，而不是继续延长纯 MSE 对齐。
4. continuation 更应从较好的中间 checkpoint 分叉，而不是从过早的点重来。

## 复用条件

下面场景可以直接复用本文流程：

- 小雷小雷 / 小雷快拍后续任一蒸馏分支的漏召回分析
- 嗨小问 / 你好问问的小模型蒸馏后“主观明显但学生不触发”的案例
- 需要判断问题更像 threshold、表征，还是数据域偏移时
