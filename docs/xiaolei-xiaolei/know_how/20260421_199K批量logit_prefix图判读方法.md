# 199K 批量 logit / prefix 图判读方法

## 适用场景

当单条 `diagnose_wav.py` 已经确认下面几件事时，下一步优先看批量图，而不是继续只盯 `report.json`：

- 学生在 `test split` 上漏召回
- 老师 `159.pt` 能收同类样本
- `threshold=0.0` 也仍有大量 `rejected`
- 想判断问题到底更像“前缀起步失败”还是“重复结构扩展失败”

当前对应脚本：

- [`tools/compare_xiaolei_miss_logits.py`](/home/wayne/work/code/project/ffalcon/wekws_baseline/tools/compare_xiaolei_miss_logits.py)

输出目录示例：

- `examples/hi_xiaowen/s0/exp/fsmn_ctc_xlxl_distill_199k/analysis/logit_compare_小雷小雷_test_119/`

## 图的构成

脚本会同时输出两类图：

- `student_teacher_curves.png`
  - 画 `<blk> / <filler> / 小 / 雷` 的 `logit` 和 `softmax prob`
- `student_teacher_prefix_curves.png`
  - 画 `小 / 小雷 / 小雷小 / 小雷小雷` 四条 CTC prefix 曲线

注意：

- 横坐标必须按 `frame_shift * frame_skip` 解释
- 当前配置是 `10ms * 3 = 30ms`
- 所以 `66` 帧输出对应约 `1.98s`，不是 `0.66s`

## 推荐判读顺序

### 1. 先看 prefix 图

优先看 `student_teacher_prefix_curves.png`。

原因：

- token 图只告诉你“某个 token 在什么时候强”
- prefix 图更接近真正关心的问题：完整关键词路径到底是在哪一段掉队的

读法：

- 如果 `小` 就明显落后老师，说明更像“前缀起步失败”
- 如果 `小` 和 `小雷` 还行，但 `小雷小` 开始掉下去，说明更像“重复结构扩展失败”
- 如果四条曲线整体都差不多，只是老师更早，才更像纯时序对齐问题

### 2. 再看 token 图

再回看 `student_teacher_curves.png`，重点只看前半段：

- `<blk>` 是否长时间压住一切
- `<filler>` 是否先起一个过渡峰
- `小` 是否紧随其后出现一个足以保住前缀的肩峰

不要只看全局峰值。

## 最容易误判的点

### 1. 图里画的是 token 类别，不区分位置

图里的 `小` 并不区分 `小1` 和 `小2`。

这意味着：

- 后半段出现一个很高的 `小` 峰，不代表 `小1` 学对了
- 它也可能只是 `小2` 强，而 `小1` 早就没保住

### 2. 某个 token 很高，不等于完整路径能保住

学生经常在后半段能把 `小` 或 `雷` 顶得很高，但最终还是只输出：

- `雷 小 雷`
- `小 雷`
- `小 小 雷`

所以真正该关心的是 prefix 曲线，而不是单个 token 的静态峰值。

## 当前 119.pt 这批图的结论

当前 `119.pt` 的 `10` 条“老师收、学生漏”代表图，可粗分为三类：

1. 重复扩展失败：`4 / 10`
   - 学生 `小`、甚至 `小雷` 已经不差，但扩不成 `小雷小 / 小雷小雷`
   - 典型输出：`小 雷`
2. 前缀起步失败：`3 / 10`
   - 学生从 `小` 开始就整体偏弱
   - 典型输出：`雷 小 雷`
3. 老师也不完全稳定：`3 / 10`
   - 老师自己也只到 `<filler> 小 雷` 或 `<filler> 小 雷 小`
   - 这类不适合作为最强监督样本

因此当前更准确的判断应是：

- 当前问题不只是“第一个 `小` 概率不够”
- 更大的平均差距其实出现在：
  - `小雷 -> 小雷小`
  - `小雷小 -> 小雷小雷`
- 主问题应理解为：
  - 前缀起步偏弱
  - 重复结构扩展失败

## 为什么老师在 0.3 秒左右总有一个 `<filler>` 峰

当前这批图里，老师经常在 `0.27s` 左右出现一个稳定 `<filler>` 峰。

这不应理解成“老师把关键词前缀识别错了”，而更像是：

- 老师先用 `<filler>` 吃掉前导非关键词区域
- 再把 `小1` 前缀托起来

原因：

1. 词表里本来就有 `<filler>`，它承担前导杂音 / 过渡语音 / 非关键词部分的统一吸收角色
2. 当前这批 wav 往往关键词在后半段，前面本来就不是 `小雷小雷`
3. CTC 会把很多连续帧的 `<filler>` 折叠成一个节点；图上看到的只是最强时刻

当前代表样本里更准确的对应关系是：

- `0.27s` 左右的大峰主成分是 `<filler>`
- `0.30s` 左右 `小` 才会出现一个较弱肩峰

因此老师真正更强的地方不是“更早就听到很高的 `小1`”，而是：

- 前导 `<filler>` 建模更稳定
- 正确 prefix 不会在最开始就被 `<blk>` 压死

而学生更常见的问题是：

- 前半段长期被 `<blk>` 压住
- 没有老师那种 `<filler> -> 小1` 的自然过渡
- 正确 prefix 还没立住，后面就更容易塌成 `雷小雷`

## 从图里该得出的训练启发

这类图支持的不是“直接给 `小` 单字加权”，而是下面这条更具体的结论：

- 真正需要保住的是：
  - `blank / filler -> 小1` 的过渡
  - `小 -> 小雷 -> 小雷小 -> 小雷小雷` 的 prefix 扩展能力

所以如果后面要改训练，优先应围绕“前缀与路径”做，而不是围绕“单个 token 分类”做。

## 复用条件

下面场景可以直接复用本文方法：

- 小雷小雷后续任一蒸馏分支的老师/学生批量对照
- 想判断漏检更像“前缀起步失败”还是“重复扩展失败”
- 想解释 `<filler>`、`<blk>`、`小 / 雷` 各自对前缀存活的作用
