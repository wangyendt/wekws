# 20260422 199K 硬正样本 replay v2 之后的下一步计划

## 1. 当前状态

`hardpos_replay_from119_v2` 已经把 replay-only 这条线最自然的一次改良尝试做完了：

- replay 更强
- schedule 更短
- lr 更低

但结果仍然是：

- 原始 `test split` 基本没动
- streaming 没有额外收益
- repeated hard case 仍然 `1/3`

因此当前不建议继续按原思路往下堆 `v3 / v4`。

## 2. 为什么不建议继续 replay-only

`v2` 的意义就在于，它已经替我们测试了下面这两个假设：

1. 是不是 replay 太弱
2. 是不是 finetune 太长

结果都是否。

如果在已经把：

- `replay_factor` 从 `3` 拉到 `10`
- `finetune_epochs` 从 `60` 缩到 `30`
- `lr` 从 `1e-4` 压到 `5e-5`

之后，hardcase 还是：

- `1/3`

那么继续把 replay-only 再堆到：

- `20`
- `15 epoch`
- `10 epoch`

收益预期已经很低。

## 3. 下一步主线

下一步更合理的方向是：

- **teacher-hit / student-miss 样本加权**

而不是继续 replay-only。

原因是：

- replay rows 复制本质上只是间接提高这批样本的训练占比
- 现在已经证明这种间接方法不够

那下一步就该切到更直接的做法：

- 在 loss 层显式给这批样本更高权重

## 4. 建议的样本加权 v1

### 4.1 数据来源

仍然沿用已经挖好的那一批：

- `826` 条 `teacher-hit / student-miss`

### 4.2 训练原则

仍然从：

- `119.pt`

起步，不从任何 `replay v1/v2` 继续接着训。

### 4.3 核心改动

在训练阶段引入：

- per-sample weight

建议首轮做法：

- 普通样本权重：`1.0`
- hard positive 权重：`4.0` 或 `5.0`

这样比复制 rows 更直接，也更容易精确控制干预强度。

### 4.4 schedule 建议

首轮仍然保持保守：

- `finetune_epochs = 30`
- `finetune_lr = 5e-5`
- `head_lr_ratio = 0.1`

这样能把变化集中在：

- sample weighting 本身

而不是同时掺进太多别的变量。

## 5. gate

这条线至少看三件事：

1. 原始 `test split`
   - `小雷小雷` 至少要明显高于 replay v2 的 `93.55%`
2. 全量 streaming
   - 不能比静态更差
3. repeated hard case
   - 至少要从 `1/3` 恢复到旧学生 `119.pt` 的 `2/3`

如果样本加权 v1 仍然过不了第三条，再考虑：

- `sample weighting + output KD`

## 6. 当前收口

当前最合理的决策是：

- 停止 replay-only 路线
- 不继续做 `v3 / v4`
- 下一步切到 `teacher-hit / student-miss` 样本加权
