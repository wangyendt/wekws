# 20260422 199K 输出蒸馏 v1 之后的下一步计划

## 1. 当前状态

第一版输出蒸馏：

- `exp/fsmn_ctc_xlxl_distill_199k_output_kd_from119_v1/avg_30.pt`

在原始 `test split` 上的结果已经出来：

- `小雷小雷 = 92.35%`
- `小雷快拍 = 99.22%`

相对当前最好学生：

- `119.pt = 96.70% / 99.61%`

因此这版没有通过主 gate。

## 2. 先怎么判断这版还有没有保留价值

虽然主 gate 没过，但这版训练的设计目标并不只是静态 accuracy，
还包括：

- 更贴近老师输出行为
- 改善 blank 到 non-blank 过渡
- 改善 repeated hard case 的第一段起步

所以当前最合理的下一步不是立刻再训一版，而是先做两件事：

1. 跑全量 streaming eval
2. 跑 repeated hard case gate

要回答的问题是：

- 这版虽然静态精度没超过 `119.pt`
- 但在 streaming / first-segment 问题上，是否有定向改善

## 3. 当前推荐顺序

### 3.1 先补这版的 streaming gate

先看：

- 原始 `test split` 的 streaming 口径
- 和 `119.pt`、`159.pt` 的差距

如果 streaming 也没有明显改善，那可以更明确地判：

- output KD alone 当前收益不足

### 3.2 再补 repeated hard case gate

优先看前面已经反复出现的几条：

- `XIAOLEI_XIAOLEI_0031_1m_f_04_fast_01_rep_10`
- `XIAOLEI_XIAOLEI_0021_1m_f_01_fast_04_rep_14`
- 以及那 10 条代表性 hard case 里的批量模式

最关键的问题不是：

- 总共触发了几次

而是：

- 是否仍然稳定漏掉第一段 `segment 0`

## 4. 根据 gate 结果怎么分叉

### 4.1 如果 streaming / hard case 有改善

如果出现这种情况：

- 原始 `test split` 虽然没超过 `119.pt`
- 但 streaming 更接近老师
- repeated hard case 漏第一段的情况减少

那么下一版应继续沿当前方向，但要叠加更强的数据干预：

1. 仍以 `119.pt` 为起点
2. 保留 output-level KL
3. 保留 blank/non-blank KD
4. 再加 hard positive replay

更准确地说，下一版应该是：

- `output KD + hard positive replay`

而不是单独再扫更多 KD 超参。

### 4.2 如果 streaming / hard case 也没有改善

如果出现这种情况：

- 原始 `test split` 没过
- streaming 也没过
- repeated hard case 的第一段问题也没缓解

那当前结论就更明确：

- output KD alone 当前收益不够

这时下一步不再优先扫 `kd_weight / blank_kd_weight / temperature`，
而应直接切到：

1. 挖 `teacher-hit / student-miss` hard positive
2. 做 hard positive replay
3. 必要时做样本加权

也就是说：

- 把主矛盾重新放回困难样本分布，而不是只放在输出蒸馏目标

## 5. 当前最推荐的下一版训练方向

在没有看到 streaming gate 之前，我当前最推荐的候选主线是：

- 仍以 `119.pt` 为起点
- 不从 `output_kd_from119_v1` 再继续接着训
- 做一版 `hard positive replay + output KD`

这样做的原因是：

- `119.pt` 仍然是当前最好学生
- `output_kd_from119_v1` 的原始 test split 已经明显掉点
- 所以下一版更稳的做法，是把 output KD 当成辅助手段，而不是把这版结果当成新的主基线

## 6. 当前不建议做的事

- 不继续沿 continuation 路线扩分支
- 不因为这版失败就直接放弃 output KD
- 不直接把 `output_kd_from119_v1` 当成新的主起点
- 不在还没看 streaming gate 之前就盲目扫更多 KD 超参

## 7. 当前收口

当前最合理的动作顺序是：

1. 先跑完 `output_kd_from119_v1` 的 streaming gate
2. 先跑完 repeated hard case gate
3. 再决定下一版是：
   - `output KD + hard positive replay`
   - 还是直接 `hard positive replay` 为主

就目前已有结果看，下一步大概率会走向：

- `119.pt` 基线
- 叠加 hard positive replay
- output KD 作为辅助，而不是唯一改动
