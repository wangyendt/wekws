# 20260422 199K 自治实验队列计划

## 1. 目标

从今天开始，把 199K 学生实验从“单次手工试验”切成“本地自治队列”：

- 自动等待当前实验结束
- 自动补 static / streaming / repeated hard case
- 自动把结果落盘
- 自动决定下一轮

这样后面就不再依赖人工盯 tmux 和手工抄结果。

## 2. 当前边界

这套自治不是无限自由搜索，而是：

- 在明确的实验空间里自动排队
- 用预设 gate 自动选下一条

这样做的好处是：

- 行为可复现
- 不会无边界盲跑
- 每轮结果都有文档和状态文件可追踪

## 3. 队列设计

### 3.1 当前起点

当前正在运行：

- `hardpos_replay_v2`

对应配置：

- `replay_factor = 10`
- `finetune_epochs = 30`
- `finetune_lr = 5e-5`

### 3.2 后续候选

如果 `v2` 仍然没有把 repeated hard case 从 `1/3` 拉回去，则继续：

1. `hardpos_replay_v3`
   - `replay_factor = 20`
   - `finetune_epochs = 15`
   - `finetune_lr = 3e-5`
2. `hardpos_replay_v4`
   - `replay_factor = 20`
   - `finetune_epochs = 10`
   - `finetune_lr = 2e-5`
   - `head_lr_ratio = 0.05`

这条队列的直觉是：

- replay-only v1 方向不是完全错
- 但信号太弱、schedule 太长
- 所以接下来优先试“更强 replay + 更短 schedule + 更小漂移”

## 4. Gate

每一轮都固定看三件事：

1. 原始 `test split`
2. 全量 streaming eval
3. 两条 repeated hard case
   - `0031`
   - `0021`

重点判断：

- `小雷小雷` 静态精度是否重新逼近 `119.pt`
- streaming 是否不劣于静态
- repeated hard case 是否至少恢复到旧学生 `119.pt` 的 `2/3`

## 5. 停止条件

### 5.1 成功停止

满足下面条件之一就停止：

- 静态精度追平或超过 `119.pt`
- 并且 repeated hard case 至少恢复到 `2/3`

### 5.2 失败收口

如果跑到 `v4` 仍然达不到：

- 说明单纯复制 replay rows 这条线基本到头

这时下一步不再继续堆 replay factor，而切到：

- `teacher-hit / student-miss` 样本加权
