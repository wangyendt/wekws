## 1. 当前目标

今天这轮计划不再继续扩散新的 from119 变体，而是先把当前 `199K` 路线收敛成一个明确结论：

1. `replay_from99` 是否能超过当前最好学生 `119.pt`

当前工作重点是“先完成最后一个低成本验证点，再进入更有针对性的训练改动”，而不是继续拍脑袋追加超参。

### 2.2 199K 主线历史结果

## 3. 当前诊断判断

已有单条诊断和集合级交叉都指向同一个方向：

- 当前很多漏检不是阈值问题
- 学生在 `threshold=0.0` 时仍然不触发
- 代表样本常塌成 `雷 小 雷`
- 更像第一段 `小雷` 没保住，而不是“整句完全没学到”

当前可以先接受下面这条判断作为工作假设：

- 主矛盾是 `199K` 学生没有稳定保住完整 `小-雷-小-雷` 路径
- 表现形式往往是：
  - 第一个 `小` 非 blank 支撑偏弱
  - 第一段 `小雷` 被 blank 吃掉
  - 最终 greedy/beam 退化成 `雷 小 雷`

因此下一步不再优先扫阈值，也不把重点放在继续延长 `FA` 对齐，而是优先统计“漏检到底在塌成什么模式”。

## 5. 跑完 replay_from99 后的决策顺序

### Step 1：先做原始 test split gate

`replay_from99` 跑完后，先只看：

- `229.pt`
- 原始 `test split`
- `analyze_exp_test_stats.py --target-fa-per-hour 1.0 --pick-mode legacy`

判断标准仍然保持简单：

1. 是否超过当前最好学生 `119.pt`
2. 是否比 `replay_from79` 更接近老师 `159.pt`

### Step 2：只有超过 119.pt 才补跑外部 test.list

如果 `replay_from99 <= 119.pt`，则：

- 不补跑外部 `test.list`
- 不继续在当前 continuation recipe 上扩新分支

如果 `replay_from99 > 119.pt`，则：

- 再补跑外部 `test.list`
- 判断它是否只是过拟合原始 `test split`

### Step 3：无论 replay_from99 结果如何，都补做漏检模式统计

接下来不只是看 accuracy 表格，而是要看漏检到底是什么形态。

已经准备好的统计脚本：

- `tools/summarize_xiaolei_miss_diagnosis.py`

它会：

1. 从 `score.txt` 找出目标关键词漏检正样本
2. 批量跑 `diagnose_wav`
3. 汇总：
   - 有多少条 greedy 直接塌成 `雷 小 雷`
   - beam top1 失败模式
   - `blank / 小 / 雷` 的相对强弱
   - 一个“疑似第一段被 blank 吃掉”的启发式计数

推荐对下面三组统一跑：

- `119.pt`
- `replay_from79/229.pt`
- `replay_from99/229.pt`

目的不是为了写更多分析，而是为了回答两个最关键问题：

1. `replay_from99` 有没有减少“塌成雷小雷”的比例
2. 失败模式是否仍然主要集中在“第一段前缀丢失”

## 6. 如果 replay_from99 仍然不超过 119.pt，下一步做什么

如果 `replay_from99` 仍然没有超过 `119.pt`，则当前应暂停继续试当前 `FA continuation` 超参，而改为进入训练干预阶段。

优先级建议如下。

### 6.1 第一优先级：hard positive replay

这是下一步最值得先做的轻量干预。

目标样本：

- 老师 `159.pt` 能收
- 学生 `119.pt` 或 `replay_from79/99` 漏掉
- 尤其是 `小雷小雷` 漏检样本

做法：

- 先从评测结果里挖出这批 hard positive
- 生成单独的 hard set
- 在 finetune 阶段对这批样本重复采样或直接复制若干倍

选择它作为第一优先级的原因：

- 不需要先大改 loss
- 最接近当前证据指向的问题
- 能直接测试“补关键正样本是否能把第一段路径拉回来”

### 6.2 第二优先级：teacher-hit / student-miss 样本加权

如果 hard positive replay 仍然不够，再进入样本加权。

做法：

- 对 teacher-hit / student-miss 这批样本在训练时给更高权重
- 权重只对困难正样本生效，不全局抬高某个 token

这里特别强调：

- 当前不建议直接做“给 `小` 这个 token 加权”
- 因为现在的问题更像“完整前缀路径保不住”，而不是单个字概率不足
- 直接抬高 `小`，有机会把包含 `小` 的非目标语音一起抬高，副作用是误唤醒

### 6.3 第三优先级：posterior / path 约束

如果前两步仍然不够，再考虑更重的 loss 级改动。

目标：

- 显式鼓励完整 `小-雷-小-雷` 路径
- 减少第一段被 blank 吃掉的情况

这类改动可能包括：

- 更直接的 posterior 约束
- 更直接的 keyword path 约束
- 或更针对第一段前缀保持的辅助目标

这一步不应在 `replay_from99` 结果出来前就直接上，因为改动成本明显更高。

## 7. 当前明确不做的事

基于今天之前已经拿到的结果，下面这些方向暂时停止：

- 不再继续沿 `distill 199K tuned` 追加更长 schedule
- 不再继续补 `ft_from119_ctc_v2`
- 不再继续扩新的 from119 continuation 变体
- 不再只靠扫阈值解释当前漏检
- 不再把“给 `小` 单字加权”当成第一反应

## 8. 本轮计划的预期产出

本轮的目标不是立即得到最终最优 `199K`，而是把问题收敛成下面两类之一：

1. `replay_from99` 成为新的最优 continuation 分支，值得继续补外部集
2. `replay_from99` 仍无法超过 `119.pt`，则当前 `FA continuation` 方向阶段性收口，转向 hard positive replay / 样本加权 / posterior-path 约束

也就是说，本轮最重要的不是“再多跑一个实验”，而是把下一阶段工作的入口从“拍脑袋调超参”，切到“围绕明确失败模式做干预”。
