# 20260422 199K 硬正样本 replay 实施方案

## 1. 为什么下一步切到 hard positive replay

到今天为止，已经可以确认：

- continuation 路线没有一条超过 `119.pt`
- output KD v1 也没有超过 `119.pt`
- streaming / repeated hard case 上，output KD v1 甚至比 `119.pt` 更差

因此当前主矛盾不应再定义成：

- “再找一个更好的 continuation 超参”
- 或者“再扫一组 KD 权重”

而应定义成：

- 困难正样本上，学生没有学会老师那种更稳的 prefix/path 行为

所以接下来最值得先做的是：

- 从 train split 里挖 `teacher-hit / student-miss` 的 hard positives
- 在训练时把这批样本重复回放

## 2. 这一步的 intuition

这一步的直觉不是“多喂一点正样本”这么简单，而是：

- 只对老师已经能收、学生还收不到的那批正样本做定向强化

也就是说，训练信号更聚焦在：

- 目前最缺的那部分正样本分布

相比继续扫 KD-only，这一步更直接回答：

- 如果把训练注意力明确拉向困难正样本，学生能不能先把召回救回来

## 3. 这一步怎么做

### 3.1 先在 train split 上各跑一次 teacher / student

目的：

- 不用 test split 做 replay，避免评测泄漏
- 直接在 train split 上挖 hard positives

### 3.2 从 train split 里抽取 hard positives

抽取条件：

- 文本是目标关键词 `小雷小雷`
- 老师 `159.pt` 能正确检测到目标关键词
- 学生 `119.pt` 不能正确检测到目标关键词

### 3.3 只增强 train split

增强方式：

- 把这批 hard positive 在 train `data.list` 里额外重复若干次

约束：

- `dev / test` 完全不动
- 只改 train split

## 4. 首轮建议配置

### 4.1 数据增强配置

首轮建议先保守一点：

- `replay_factor = 3`

意思是：

- 每条 hard positive 额外再重复 `3` 次

### 4.2 训练配置

训练起点仍然固定：

- `exp/fsmn_ctc_xlxl_distill_199k/119.pt`

首轮先不要再叠 output KD，先验证 replay 本身的收益：

- `finetune_kd_weight = 0.0`
- `finetune_blank_kd_weight = 0.0`

其它 finetune 参数先保持和前面 from119 系列接近：

- `align_epochs = 120`
- `finetune_epochs = 60`
- `finetune_lr = 1e-4`
- `head_lr_ratio = 0.1`
- `finetune_mse_weight_start = 0.3`
- `finetune_mse_weight_end = 0.1`

## 5. 首轮 gate

这版至少要过三道 gate：

1. 原始 `test split`
   - `小雷小雷` 必须超过 `119.pt`
2. 全量 streaming eval
   - 不能比 `119.pt` 更差
3. repeated hard case
   - 不能继续稳定漏第一段

## 6. 如果首轮有效，下一步怎么接

如果 hard positive replay 首轮有效，再按这个顺序继续：

1. 先小幅扫 `replay_factor`
2. 再考虑叠加 output KD
3. 只有前两步还不够，再考虑样本加权

也就是说：

- 下一版更合理的组合是 `hard positive replay + output KD`
- 不是直接回到 KD-only

## 7. 如果首轮无效，怎么解释

如果 hard positive replay 首轮仍然没有明显改善，则更可能说明：

- 问题不只是训练样本覆盖不够
- 还需要更直接的样本加权或路径级约束

这时下一步应切到：

- teacher-hit / student-miss 样本加权

而不是继续单独扫 replay factor。

## 8. 当前收口

当前最合理的下一步主线是：

1. 在 train split 上挖 hard positives
2. 做一版 replay-only 实验
3. 用它判断：
   - 问题主要在数据分布
   - 还是必须进入更重的 loss 设计
