# 20260422 199K 硬正样本 replay v1 之后的下一步计划

## 1. 当前状态

第一版 `hard positive replay` 已经完成，结果是：

- 原始 `test split`：`93.48% / 99.35%`
- 全量 streaming：同样是 `93.48% / 99.35%`
- 两条代表性 repeated hard case：都只有 `1/3`

因此这版没有通过主 gate，也没有救回部署相关的 repeated hard case。

## 2. 当前最合理的解释

### 2.1 首轮 replay 强度太弱

这版 hard positive 挖掘结果是：

- `826` 条 hard positives
- `replay_factor=3`
- 只额外增加 `2478` 条训练样本

占比只有：

- 总 train 的约 `1.55%`

这说明：

- replay 信号本身是存在的
- 但它还远远不够强

### 2.2 长 schedule 的 finetune 漂移仍然过强

这版依然沿用了较长的 `from119` finetune：

- `finetune_epochs = 60`

而前面几轮已经反复说明：

- 只要从 `119.pt` 往后继续训太久，整体上就容易越来越偏离当前最好学生

所以这版更像是：

- replay 信号太弱
- 漂移太强

结果就是：

- 静态指标有一点回升
- 但 repeated hard case 仍然掉得厉害

## 3. 下一步不该怎么做

当前不建议的路线：

- 不从 `hardpos_replay_from119_v1` 再继续接着训
- 不继续维持 `replay_factor=3 + finetune 60 epoch`
- 不先回去扫 `KD-only`

原因很直接：

- 当前失败更像“干预不够强 + schedule 太长”
- 不是 replay 方向本身已经被证伪

## 4. 下一步建议的主线

下一步先做一版：

- **更强 replay + 更短 schedule**

也就是：

1. 仍然从 `119.pt` 出发
2. 仍然先不叠 output KD
3. 提高 replay 强度
4. 明显缩短 finetune 长度
5. 同时把学习率再压低一点，减少整体漂移

## 5. 建议的 v2 配置

### 5.1 数据配置

把 replay 强度从：

- `replay_factor = 3`

提高到：

- `replay_factor = 10`

这样 hard positive 的新增样本数会从：

- `2478`

提升到：

- `8260`

这时占总 train 的比例大约就是：

- `5.17%`

这比首轮的 `1.55%` 更像一次真正有力度的定向干预。

### 5.2 训练配置

从 `119.pt` 起步，建议改成：

- `finetune_epochs = 30`
- `finetune_lr = 5e-5`
- `head_lr_ratio = 0.1`
- `finetune_mse_weight_start = 0.3`
- `finetune_mse_weight_end = 0.1`

为什么是这组配置：

- `30` epoch 比 `60` epoch 更容易控制漂移
- `5e-5` 比 `1e-4` 更保守
- 保持其它项不变，可以更干净地回答：
  - 这次提升究竟来自“更强 replay + 更短 schedule”
  - 还是别的超参变化

## 6. v2 的 gate

这版继续不继续，至少看三件事：

1. 原始 `test split`
   - `小雷小雷` 要明显高于 v1 的 `93.48%`
   - 最好重新逼近 `95%+`
2. 全量 streaming
   - 不允许比静态更差
3. repeated hard case
   - 至少要从当前的 `1/3` 恢复到旧学生 `119.pt` 的 `2/3` 水平

如果这三个 gate 里第三条仍然过不了，就说明：

- 单纯 replay rows 复制还不够

那下一步就应切到：

- `teacher-hit / student-miss` 样本加权

而不是继续堆更高的 replay factor。

## 7. 当前收口

当前最合理的下一步不是：

- 再做一轮 KD-only
- 或者继续沿用当前 replay v1 配置

而是：

1. 回到 `119.pt`
2. 做一版 `hardpos replay v2`
3. 特点是：
   - replay 更强
   - schedule 更短
   - 学习率更低

如果这版还救不回来，再进入样本加权路线。
