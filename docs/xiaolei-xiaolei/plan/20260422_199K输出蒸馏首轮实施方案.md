# 20260422 199K 输出蒸馏首轮实施方案

## 1. 这次实现了什么

今天已经把训练代码改到可以直接做第一版“贴近老师输出行为”的实验。

本次改动不再只做：

- block feature MSE
- student 自己的 CTC

而是在 finetune 阶段额外加入两项蒸馏：

- output-level KL
- blank / non-blank KD

当前 finetune 总 loss 变成：

- `L = w_mse * L_mse + (1 - w_mse) * L_ctc + λ_kd * L_kl + λ_blank * L_blank`

其中：

- `L_kl`：teacher / student 最终输出 logits 经过 softmax 后的 KL
- `L_blank`：teacher / student 在 blank-vs-nonblank 上的辅助蒸馏

实现文件：

- `wekws/utils/executor_distill.py`
- `wekws/bin/train_distill.py`
- `examples/hi_xiaowen/s0/run_distill.sh`

## 2. 这次没有做什么

这次故意没有直接做这些更重的改动：

- 不强行把 `<filler>` 写进正样本标签
- 不直接做 prefix/path loss
- 不直接做 hard positive replay

原因是这次先验证一个更基础的问题：

- 如果让学生直接学老师的最终输出分布和 blank 释放节奏，能不能先把 `119.pt` 往上拉

## 3. 为什么这版值得先跑

当前证据已经比较一致：

- continuation 三条 seed 全部失败
- 批量 `logit / prefix` 图说明学生没有把老师输出行为学到位
- repeated hard case 流式实验说明老师能稳定多次触发，学生不行

所以现在最该先验证的是：

- 只改训练目标，不改数据，不改词表，不改部署逻辑
- 看 student 是否能更贴近 teacher 的输出行为

## 4. 首轮实验固定设置

首轮先不要从头训，直接从当前最好学生起步：

- 起点 checkpoint：`exp/fsmn_ctc_xlxl_distill_199k/119.pt`

训练策略：

- 不再重新走对齐阶段
- 用 `align_epochs=120`，让恢复训练从 epoch `120` 直接进入 finetune
- 新增 finetune 60 个 epoch，跑到 `179.pt`

首轮建议超参：

- `finetune_lr = 1e-4`
- `head_lr_ratio = 0.1`
- `finetune_mse_weight_start = 0.3`
- `finetune_mse_weight_end = 0.1`
- `finetune_kd_weight = 0.2`
- `finetune_blank_kd_weight = 0.05`
- `kd_temperature = 2.0`

这版的目标不是一步到位，而是先看：

- 原始 `test split` 是否超过 `119.pt`
- 全量 streaming 是否更接近老师
- repeated hard case 的第一段漏检是否减少

## 5. 首轮 gate

这版训练出来以后，至少要过下面三道 gate：

1. 原始 `test split`
   - `小雷小雷` 必须超过 `119.pt`
   - `小雷快拍` 不能明显拉坏
2. 全量 streaming eval
   - 流式口径下的 `小雷小雷` 要比 `119.pt` 更接近老师
3. repeated hard case
   - 至少检查现在那几条 `x3` hard case
   - 看学生是否比 `119.pt` 更不容易漏第一段

## 6. 如果首轮有效，下一步怎么接

如果首轮有效，再按这个顺序继续：

1. 把 `λ_kd / λ_blank / temperature` 做小范围 sweep
2. 再叠加 hard positive replay
3. 只有前两步还不够，再考虑 prefix/path 约束

## 7. 如果首轮无效，怎么解释

如果这版仍然明显不如老师，优先这样解释：

- 只靠输出蒸馏还不够
- 学生不仅输出行为没学到，困难样本分布本身也需要更强干预

这时下一步再切到：

- hard positive replay
- teacher-hit / student-miss 样本加权

而不是回到 continuation 路线。
