# 20260423 199K 样本加权 v1 实施方案

## 1. 背景

到 `hardpos_replay_from119_v2` 为止，replay-only 这条线已经基本收口：

- `replay_factor` 从 `3` 提到 `10`
- `finetune_epochs` 从 `60` 缩到 `30`
- `finetune_lr` 从 `1e-4` 降到 `5e-5`

但结果仍然是：

- 原始 `test split` 只有轻微波动
- 全量 streaming 与静态完全一致，没有额外收益
- 两条最关键 repeated hard case 仍然只有 `1/3`

这说明：

- 复制 rows 这种“间接提高 hard positive 占比”的办法已经不够

下一步应改成更直接的干预：

- **teacher-hit / student-miss 样本加权**

## 2. 本轮 intuition

这轮不是再去改变数据分布本身，而是：

- 保持原始训练集不变
- 只在 loss 层显式提高 hard positive 的梯度权重

这样相对于 replay-only，有两个优点：

1. 不会继续扩大 train list 本身，避免重复样本过多带来的分布漂移
2. 可以更精确地控制干预强度，例如：
   - 普通样本 `1.0`
   - hard positive `5.0`

如果这条线仍然没有明显改善，再考虑：

- `sample weighting + output KD`

而不是继续单独堆 replay。

## 3. 数据来源

仍然沿用已经挖好的这一批 hard positives：

- 关键词：`小雷小雷`
- 条件：`teacher-hit / student-miss`
- 数量：`826`

这批样本来自：

- 老师：`exp/fsmn_ctc_xlxl_top20_weight_surgery/train_159/score.txt`
- 学生：`exp/fsmn_ctc_xlxl_distill_199k/train_119/score.txt`

## 4. 训练设置

### 4.1 起点

仍然从：

- `exp/fsmn_ctc_xlxl_distill_199k/119.pt`

起步，不从任何 `replay v1/v2` 或 `output_kd v1` 继续接着训。

### 4.2 核心设置

首轮采用：

- 普通样本权重：`1.0`
- hard positive 权重：`5.0`

### 4.3 schedule

继续保持保守，尽量只验证“样本加权”这一项：

- `align_epochs = 120`
- `finetune_epochs = 30`
- `finetune_lr = 5e-5`
- `head_lr_ratio = 0.1`
- `finetune_mse_weight_start = 0.3`
- `finetune_mse_weight_end = 0.1`
- 不叠 `output KD`

## 5. 代码实现原则

这轮不改外部 `wenet` 包源码，而是在当前仓库内实现：

1. 用脚本生成 `key -> weight` 的 JSON 映射
2. 在 `train_distill.py` 里读取这个映射
3. 在 `executor_distill.py` 里根据 batch 的 `keys` 查权重
4. 对以下 loss 都做 weighted mean：
   - CTC
   - block MSE
   - output KD
   - blank/nonblank KD

这样后面如果要做：

- weighting-only
- weighting + KD

都可以直接复用。

## 6. Gate

这轮至少看三道 gate：

1. 原始 `test split`
   - `小雷小雷` 至少要明显高于 `93.55%`
2. 全量 streaming
   - 不能比静态更差
3. repeated hard case
   - 至少要从当前 `1/3` 恢复到旧学生 `119.pt` 的 `2/3`

如果前两条有改善，但第三条还是 `1/3`，说明这条线仍然没有真正解决部署相关问题。

## 7. 成功 / 失败后的下一步

### 7.1 如果 v1 明显改善

下一步可以尝试：

- 维持 sample weighting
- 低权重叠加 `output KD`

### 7.2 如果 v1 仍无明显改善

下一步不再继续堆：

- `hard_weight = 8/10`

而应直接转到：

- `sample weighting + output KD`

因为那时说明：

- 单独加大 hard positive 的监督强度仍然不够
- 还需要进一步约束学生的输出行为
