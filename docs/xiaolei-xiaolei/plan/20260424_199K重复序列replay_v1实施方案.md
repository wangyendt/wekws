# 20260424 199K 重复序列 replay v1 实施方案

## 1. 背景判断

当前最佳学生仍是：

- `head-only hardpos replay v1 avg_10.pt`
- 原始 test split：`98.20% / 99.74%`
- `0021 x3 = 3/3`
- `0031 x3 = 1/3`

已经验证无效或收益不足的方向：

- 继续拉长 head-only epoch：无新增收益
- 只筛 prefix-start 子集再强 replay：`98.05% / 99.74%`，`0031 x3` 仍 `1/3`
- `gap=300ms` 只能让 `129.pt` 到 `2/3`，老师仍 `3/3`

因此问题不只是“短样本重复次数不够”，也不只是无间隔拼接边界，而是学生没有稳定学到连续多次 `小雷小雷` 的 CTC 路径。

## 2. 实验假设

把 hard positive 直接拼成长序列，并把标签也扩展为多遍关键词，可以给学生更明确的路径约束：

- 输入音频：`wav + wav + wav`
- 标签：`小 雷 小 雷 小 雷 小 雷 小 雷 小 雷`
- 训练范围仍然只动 head，避免破坏已有 backbone 表征

这比单纯把同一条短样本复制多次更直接，因为 CTC loss 会在一条长音频里学习三段完整关键词路径。

## 3. 数据方案

使用新脚本：

- `tools/build_xiaolei_repeated_sequence_replay_data.py`

首版数据：

- 基础数据：`data_xlxl_0327_ctc_v1_clean_hardpos_replay_v2`
- 额外样本来源：`exp/xlxl_aux/prefixstart_hardpos_from119_v1_rows.jsonl`
- 选中样本：`513`
- `repeat=3`
- `gap_ms=0`
- `add_factor=1`
- 输出数据：`data_xlxl_0327_ctc_v1_clean_repeatedseq_replay_v1`

已生成结果：

- base train rows：`167881`
- added rows：`513`
- augmented train rows：`168394`
- label example：`小 雷 小 雷 小 雷 小 雷 小 雷 小 雷`

## 4. 训练方案

实验目录：

- `exp/fsmn_ctc_xlxl_distill_199k_repeatedseq_replay_head_only_from119_v1`

配置：

- 起点：`exp/fsmn_ctc_xlxl_distill_199k/119.pt`
- 训练范围：`finetune_trainable_scope=head_only`
- `finetune_epochs=10`
- `finetune_lr=1e-4`
- `head_lr_ratio=1.0`
- `num_average=10`

## 5. 通过标准

主 gate：

- 原始 test split 不低于 `98.20% / 99.74%`
- 或者 `0031 x3` 从 `1/3` 提升到 `2/3` 或 `3/3`

如果静态掉到 `98.05%` 以下且 `0031` 不动，则重复序列 replay v1 判失败。

## 6. 后续分支

如果 v1 有收益：

- 尝试 `gap_ms=300` 的 repeated-sequence 数据
- 尝试 `add_factor=2`
- 对比 `avg_10.pt` 与最后单 checkpoint

如果 v1 无收益：

- 停止数据 replay 方向
- 转向更显式的 path/posterior 约束，例如对 teacher 对齐出的 token 峰值附近加局部 token CE
