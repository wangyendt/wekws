# 20260421 小雷 199K 图像诊断后的训练优先级

## 1. 这份文档回答什么问题

这份补充计划只回答下面三个问题：

1. 批量 `logit / prefix` 图给当前 `199K` 路线带来了什么新结论
2. 下一步应该优先改训练、改数据增强，还是继续扫 continuation 超参
3. 如果要做 hard positive replay / 样本加权 / path 约束，应该先围绕哪类失败模式做

当前输入证据来自：

- `119.pt` 的 `10` 条“老师收、学生漏”代表图
- 路径：`examples/hi_xiaowen/s0/exp/fsmn_ctc_xlxl_distill_199k/analysis/logit_compare_小雷小雷_test_119/`

## 2. 图像诊断带来的新结论

### 2.1 当前问题不只是“第一个小概率低”

这批图说明：

- `小` 的问题确实存在
- 但更大的平均差距其实出现在：
  - `小雷 -> 小雷小`
  - `小雷小 -> 小雷小雷`

因此当前主问题更准确的描述应是：

- 前缀起步偏弱
- 重复结构扩展失败

而不是单纯：

- 第一个 `小` 没起来

### 2.2 当前 10 条代表图的失败分型

当前可粗分为三类：

1. 重复扩展失败：`4 / 10`
   - 学生 `小`、甚至 `小雷` 已经不差，但扩不成完整重复词
   - 常见输出：`小 雷`
2. 前缀起步失败：`3 / 10`
   - 学生从 `小` 开始就整体偏弱
   - 常见输出：`雷 小 雷`
3. 老师也不完全稳定：`3 / 10`
   - 老师自己也只到 `<filler> 小 雷` 或 `<filler> 小 雷 小`
   - 这类不应作为最强监督样本

### 2.3 对老师 `<filler>` 峰的解释

老师在 `0.27s` 左右出现的稳定 `<filler>` 峰，应理解成：

- 老师先用 `<filler>` 吃掉前导非关键词区域
- 再把 `小1` 前缀托起来

因此当前老师更强的地方不是“更早给出了一个特别高的 `小1`”，而是：

- 前导 `<filler>` 建模更稳定
- 正确 prefix 不会在最开始就被 `<blk>` 完全压死

学生更常见的问题则是：

- 前半段一直停留在 `<blk>` 主导状态
- 没有老师那种 `<filler> -> 小1` 的自然过渡
- 后面更容易塌成 `雷小雷` 或 `小雷`

## 3. 下一步应该怎么优化学生模型

当前优先级应明确为：

1. 先做训练侧干预
2. 再考虑定向数据增强
3. 不把泛化数据增强放在第一优先级

原因很简单：

- 同一批数据上老师已经能收
- 说明样本本身并不缺失
- 当前更像学生容量 / 优化 / 路径保持问题，而不是“没见过这种数据”

## 4. 具体优先级

### 4.1 第一优先级：hard positive replay

这是当前最值得先做的轻量干预。

目标样本：

- 老师 `159.pt` 能稳定形成完整 `<filler> 小 雷 小 雷`
- 学生 `119.pt` 或后续分支漏掉
- 尤其聚焦 `小雷小雷`

建议先分两桶：

1. 前缀起步失败桶
   - 典型输出：`雷 小 雷`
2. 重复扩展失败桶
   - 典型输出：`小 雷` / `小 小 雷`

暂时排除：

- 老师自己也只到 `<filler> 小 雷` 或 `<filler> 小 雷 小` 的边界样本

理由：

- 不需要先改 loss
- 与当前图像证据最匹配
- 能直接验证“补关键正样本是否能把前缀起步与重复扩展一起拉回来”

### 4.2 第二优先级：teacher-hit / student-miss 样本加权

如果 hard positive replay 仍然不够，再做样本加权。

这里特别强调：

- 不建议直接做“给 `小` 单字加权”
- 因为当前问题更像“完整前缀路径保不住”
- 直接抬高 `小`，有机会把包含 `小` 的非目标语音一起抬高，副作用是误唤醒

更合理的方向是：

- 提高“老师稳定完整触发、学生漏掉”的 hard positive 权重
- 优先强化：
  - `blank / filler -> 小1` 的过渡
  - `小雷 -> 小雷小 -> 小雷小雷` 的路径保持

### 4.3 第三优先级：posterior / path 约束

如果前两步仍然不够，再考虑更重的 loss 级改动。

当前更建议围绕四条 prefix 做，而不是直接围绕单个 token：

- `小`
- `小雷`
- `小雷小`
- `小雷小雷`

原因：

- 当前图像诊断已经说明，真正坏掉的通常是 prefix 扩展
- 而不是单个 `小` 或 `雷` 的静态可分性

## 5. 数据增强应该放在哪

当前不建议把“泛化数据增强”放在最前面。

不推荐的第一反应：

- 先上更强噪声增强
- 先上更重 SpecAug
- 先指望泛化增强自动修复 `雷小雷`

如果要做数据增强，也应只做定向增强，而且放在 hard positive replay 之后：

- 随机前导 `filler / silence / noise` 拼接
- 随机平移关键词在 `2s` 窗口里的位置
- 强化 `<filler> -> 小1` 的过渡学习

## 6. 当前明确不建议做的事

- 不再把主要精力放在扫阈值上
- 不把“给 `小` 单字加权”当成第一反应
- 不把泛化噪声增强当成当前第一优先级
- 不在图像诊断已经给出失败模式后，还继续拍脑袋扩 continuation 超参

## 7. 这份计划如何使用

当前最合理顺序是：

1. 先跑完 `replay_from99`
2. 判断它是否仍不超过 `119.pt`
3. 如果仍不超过，则进入 hard positive replay
4. hard positive 的清单按“前缀起步失败 / 重复扩展失败”两桶维护
5. 只有前两步仍然不够，再进入样本加权或 path 约束
