# 20260402 小雷专用 top20 权重手术方案

## 结论先行

小雷下一步应当复用仓库里现成的“缩小词表 + weight surgery”链路，但词表排序文件不能继续沿用历史全局排序文件。

更稳的做法是：

1. 先基于当前 `data_xlxl_0327_ctc_v1_clean` 重新生成一份小雷专用排序文件。
2. 再用现有 `gen_reduced_dict.py` 生成 `dict_top20_xlxl`。
3. 再从 `exp/fsmn_ctc_xlxl_0327_clean_baseline_2599/79.pt` 做 weight surgery。
4. 后续蒸馏、小模型训练、评测、导出都统一复用这份新 dict。

这条路径比“在旧排序文件里强行插入 `小/雷/快/拍`”更干净，也更兼容后续脚本。

继续做小雷 top20 weight surgery。

- 没必要再走“强行保留”这条临时方案
- 直接用小雷专用排序文件生成 top20 即可

## 与后续蒸馏的兼容性

这一步要特别注意：后面无论是 weight surgery、小模型蒸馏、评测、还是导出，都应当统一使用同一套学生词表。

建议固定为：

- 教师 full dict：`examples/hi_xiaowen/s0/dict`
- 学生 top20 dict：`examples/hi_xiaowen/s0/dict_top20_xlxl`
- 学生排序文件：`examples/hi_xiaowen/s0/dict/model_vocab_freq_asr_sorted_xlxl_clean.txt`

这样后续脚本就有稳定约定：

- surgery 时：`--checkpoint_dict dict --dict_dir dict_top20_xlxl`
- 小模型训练/评测时：统一 `--dict_dir dict_top20_xlxl`
- 如果后面扩成 `top50/top100`，仍然沿用同一个小雷排序文件，只改 `--num_keywords`

也就是说，本次新生成的排序文件和 `dict_top20_xlxl` 应视为后续压缩链路的正式输入，而不是一次性临时产物。

## 推荐执行顺序

### Step 1：生成并检查小雷专用排序文件

```bash
python tools/build_sorted_vocab_from_data.py --data-dir data_xlxl_0327_ctc_v1_clean --dict dict/dict.txt --output dict/model_vocab_freq_asr_sorted_xlxl_clean.txt
```

### Step 2：生成 `dict_top20_xlxl`

```bash
python tools/gen_reduced_dict.py --sorted_file "$(pwd)/dict/model_vocab_freq_asr_sorted_xlxl_clean.txt" --num_keywords 20 --output_dir "$(pwd)/dict_top20_xlxl"
```

### Step 3：从 `79.pt` 做 top20 weight surgery

```bash
bash run_fsmn_ctc.sh --data_dir data_xlxl_0327_ctc_v1_clean --checkpoint exp/fsmn_ctc_xlxl_0327_clean_baseline_2599/79.pt --checkpoint_dict dict --target_exp_dir exp/fsmn_ctc_xlxl_top20_weight_surgery --gpus 0,1,2,3 --dict_dir dict_top20_xlxl --num_keywords 20 --checkpoint_strict false 2 2
```

### Step 4：评测原始 test split

```bash
bash evaluate.sh --checkpoint exp/fsmn_ctc_xlxl_top20_weight_surgery/79.pt --dataset test --data_dir data_xlxl_0327_ctc_v1_clean --dict_dir dict_top20_xlxl --keywords "小雷小雷,小雷快拍" --gpu 0
python examples/hi_xiaowen/s0/analyze_exp_test_stats.py --exp-dir examples/hi_xiaowen/s0/exp/fsmn_ctc_xlxl_top20_weight_surgery --test-id 79 --target-fa-per-hour 1.0 --pick-mode legacy
```

### Step 5：评测外部 test.list

```bash
python tools/prepare_xiaolei_eval_data.py --input-list /home/xushang/kws_ws/wekws/examples/hi_xiaowen/s0/datasets/test.list --output-dir codex_artifacts/xlxl_external_test_eval_max20 --max-duration 20.0 --keep-source-manifest
bash evaluate.sh --checkpoint exp/fsmn_ctc_xlxl_top20_weight_surgery/79.pt --dataset test --data_dir codex_artifacts/xlxl_external_test_eval_max20 --dict_dir dict_top20_xlxl --keywords "小雷小雷" --result_suffix xlxl_external_test_top20_79_max20 --gpu 2 --batch_size 32
python examples/hi_xiaowen/s0/analyze_exp_test_stats.py --exp-dir examples/hi_xiaowen/s0/exp/fsmn_ctc_xlxl_top20_weight_surgery --test-id test_79_xlxl_external_test_top20_79_max20 --target-fa-per-hour 1.0 --pick-mode legacy
```

## 当前建议

所以，下一步不是“默认 top20”，也不是“旧排序文件强插 token 的 forced top20”，而是：

1. 用当前 clean 数据生成小雷专用排序文件。
2. 基于该排序文件生成 `dict_top20_xlxl`。
3. 以 `79.pt` 为起点做 top20 weight surgery。
4. 同时看原始 test split 和外部 test.list。
5. 只有这一步不明显掉点，才继续做更小 backbone 的蒸馏。
