# 20260422 199K 硬正样本 replay v1 结果记录

## 1. 这组实验的 intuition

这次 `hardpos_replay_from119_v1` 的核心直觉是：

- 当前 `199K` 学生真正掉点的地方，不像是“没见过足够多正样本”
- 更像是“老师能收、学生还收不到”的那一小撮困难正样本没有被充分学会

所以这版先不继续做 `KD-only`，而是转成更直接的数据层干预：

- 从 train split 里挖 `teacher-hit / student-miss` 的 `小雷小雷` hard positives
- 把它们额外重复回放到训练里

希望验证的问题是：

- 如果只对这批最有信息量的困难正样本做定向强化，学生的召回能不能先回升

## 2. 这次具体做了什么

### 2.1 hard positive 挖掘结果

命令：

```bash
python tools/build_xiaolei_hard_positive_replay_data.py \
  --source-data-dir data_xlxl_0327_ctc_v1_clean \
  --teacher-score exp/fsmn_ctc_xlxl_top20_weight_surgery/train_159/score.txt \
  --student-score exp/fsmn_ctc_xlxl_distill_199k/train_119/score.txt \
  --keyword 小雷小雷 \
  --replay-factor 3 \
  --output-dir data_xlxl_0327_ctc_v1_clean_hardpos_replay_v1
```

挖掘结果：

- train 总样本：`159621`
- `小雷小雷` 正样本：`25109`
- hard positives：`826`
- `replay_factor=3` 后新增样本：`2478`
- 增强后 train 总样本：`162099`

这几个数字的含义是：

- `826 / 25109 ≈ 3.29%`
  说明真正落在“老师收、学生漏”的 train 正样本比例并不高
- `2478 / 159621 ≈ 1.55%`
  说明首轮 replay 对整套训练分布的扰动其实很弱

### 2.2 训练配置

起点：

- `exp/fsmn_ctc_xlxl_distill_199k/119.pt`

训练原则：

- 只验证 replay 本身，不叠 output KD

实验目录：

- `exp/fsmn_ctc_xlxl_distill_199k_hardpos_replay_from119_v1`

## 3. 结果

### 3.1 原始 test split

`avg_30.pt` 统一口径结果：

- `小雷小雷 = 93.48%`
- `小雷快拍 = 99.35%`

对比：

- `119.pt = 96.70% / 99.61%`
- `output_kd_from119_v1 = 92.35% / 99.22%`

所以这版结果可以概括成：

- 比 `output KD v1` 有所回升
- 但仍明显低于当前最好学生 `119.pt`

### 3.2 全量 streaming eval

全量 streaming 口径结果与原始 `test split` 完全一致：

- `小雷小雷 = 93.48%`
- `小雷快拍 = 99.35%`

这说明：

- 这版没有出现“静态一般，但 streaming 特别好”的隐藏收益

### 3.3 repeated hard case

代表样本：

- `XIAOLEI_XIAOLEI_0031_1m_f_04_fast_01_rep_10`
- `XIAOLEI_XIAOLEI_0021_1m_f_01_fast_04_rep_14`

两条的 `x3` streaming 结果都一样：

- 学生只触发 `1` 次
- 且都只落在 `segment 1`

对比：

- 老师仍然是标准 `3/3`
- 旧学生 `119.pt` 至少还能做到 `2/3`，覆盖 `segment 1 / 2`

所以这版不是只“静态没超过 `119.pt`”，而是：

- 在部署更相关的 repeated hard case 上，比 `119.pt` 更差

## 4. 这些结果代表什么

### 4.1 replay-only 方向不是完全没价值

和 `output_kd_from119_v1` 相比，这版至少把指标从：

- `92.35% / 99.22%`

拉回到了：

- `93.48% / 99.35%`

说明：

- 直觉上“把注意力重新拉回 hard positives”并不是完全错误

### 4.2 但首轮干预明显不够强

从数据分布上看，这版新增样本只有：

- `2478` 条
- 约占总 train 的 `1.55%`

这很可能太弱，不足以在 `from119` 的 60 epoch finetune 里稳定压过整体训练漂移。

### 4.3 主矛盾仍然没被解决

如果这版已经抓住主矛盾，至少应该看到下面之一：

- 原始 `test split` 接近或超过 `119.pt`
- streaming 总量有明显改善
- repeated hard case 从 `1/3` 恢复到至少 `2/3`

现在三条都没有出现。

尤其 repeated hard case 上，它和 `output KD v1` 一样，仍然只有：

- `1` 次触发
- 只落在 `segment 1`

所以当前更像：

- replay 信号太弱
- 训练漂移仍然过强
- prefix/path 的部署问题没有被真正打中

## 5. 这版实验需要落盘哪些结果

这版建议至少落盘这四类结果：

1. hard positive 挖掘摘要
   - `826` 条 hard positives
   - `replay_factor=3`
   - `added_rows=2478`
2. 原始 `test split`
   - `小雷小雷 93.48%`
   - `小雷快拍 99.35%`
3. 全量 streaming eval
   - 与原始 `test split` 一致
4. 两条 repeated hard case
   - `0031 x3`：学生 `1/3`，仅 `segment 1`
   - `0021 x3`：学生 `1/3`，仅 `segment 1`

## 6. 当前结论

`hardpos_replay_from119_v1` 可以判成：

- 比 `output_kd_from119_v1` 略好
- 但仍未通过主 gate
- 且在 repeated hard case 上明显不如 `119.pt`

因此它不能作为新的主基线。

更合理的解释是：

- 首轮 replay-only 的方向有一定对症性
- 但当前配置下，回放强度太弱、schedule 太长
- 还不足以真正修复 `199K` 学生的 prefix/path 问题
