# 20260422 199K 输出蒸馏 v1 完整结论

## 1. 这组实验的 intuition 是什么

这次 `output_kd_from119_v1` 的直觉不是继续沿 continuation 路线调超参，而是换一个更直接的问题去验证：

- 旧的 `199K` 学生主要学到了老师的中间层 block feature
- 但没有直接学到老师的最终输出行为

具体来说，前面的单条和批量分析已经反复说明：

- 老师更早从 `<blk>` 释放出来
- 老师更容易保住 `小 -> 小雷 -> 小雷小 -> 小雷小雷` 这条路径
- 学生更容易塌成 `雷小雷`、`小雷` 或 `小小雷`
- repeated hard case 下，旧学生 `119.pt` 对代表性 hard case 会稳定漏掉第一段

所以这次的训练直觉是：

1. 让学生不仅学 block feature
2. 还直接学老师的输出分布
3. 再额外约束 blank / non-blank 过渡

因此这版训练在 finetune 阶段引入了：

- output-level KL
- blank / non-blank KD

希望解决的问题是：

- 不是简单提升单 token 峰值
- 而是让学生更像老师那样形成稳定的输出路径和多次流式触发行为

## 2. 这次具体做了什么

起点：

- `exp/fsmn_ctc_xlxl_distill_199k/119.pt`

新增训练目标：

- `L = w_mse * L_mse + (1 - w_mse) * L_ctc + λ_kd * L_kl + λ_blank * L_blank`

关键超参：

- `finetune_kd_weight = 0.2`
- `finetune_blank_kd_weight = 0.05`
- `kd_temperature = 2.0`

实验目录：

- `exp/fsmn_ctc_xlxl_distill_199k_output_kd_from119_v1`

## 3. 结果

### 3.1 原始 test split

`avg_30.pt` 的统一口径结果：

- `小雷小雷 = 92.35%`
- `小雷快拍 = 99.22%`

对比当前最好学生：

- `119.pt = 96.70% / 99.61%`

所以静态主 gate 没过。

### 3.2 全量 streaming eval

这版 streaming 口径结果和原始 `test split` 的结果没有出现额外改善：

- `小雷小雷 = 92.35%`
- `小雷快拍 = 99.22%`

因此至少在总体指标上，没有出现“静态差一点，但 streaming 明显更好”的情况。

### 3.3 repeated hard case

对两条最典型的 repeated hard case：

- `XIAOLEI_XIAOLEI_0031_1m_f_04_fast_01_rep_10`
- `XIAOLEI_XIAOLEI_0021_1m_f_01_fast_04_rep_14`

这版学生在 `x3` streaming 下都只剩：

- `1` 次触发
- 且都只落在 `segment 1`

而老师仍然是：

- `3/3`
- 标准覆盖 `segment 0 / 1 / 2`

对比旧学生 `119.pt`：

- 旧学生虽然也失败，但至少通常还是 `2/3`
- 触发落在 `segment 1 / 2`

所以这版 repeated hard case 的结果不是“没有改善”，而是：

- 比 `119.pt` 更差

## 4. 这些结果代表什么

### 4.1 output KD 的方向不是完全错，但单独使用不够

这版比最差的几个 continuation 分支还是好一些，说明：

- “让学生贴近老师输出行为”这个方向本身未必错

但它仍明显低于：

- `119.pt`
- `replay_from79`

所以当前更准确的结论是：

- output-level KD 单独做，不够

### 4.2 它没有解决主矛盾

如果这版真的抓住了主矛盾，至少应该看到下面之一：

- 原始 `test split` 明显改善
- streaming 口径明显改善
- repeated hard case 的第一段漏检减少

现在三条都没有发生。

尤其 repeated hard case 上，它从旧学生的 `2/3` 退化到 `1/3`，说明：

- 它没有把 prefix/path 问题救回来
- 反而可能让模型更保守，导致多次流式触发能力更差

### 4.3 问题不在单 token 峰值，而在路径与分段触发

这次 repeated hard case 的学生结果里，有一个很典型的现象：

- `小` 和 `雷` 的单 token 概率峰值并不低
- 但 streaming 仍然只触发一次

这说明问题不是：

- “学生根本不会出 `小` / `雷`”

而更像：

- 这些峰值没有被组织成正确的分段 prefix/path
- 多次出现的关键词实例之间，第一段和后续段的起步/保持能力都不稳

### 4.4 `<filler>` 仍然不是直接目标

这次实验也继续支持之前那个判断：

- 老师强，不是因为“每段都要打出 filler”
- 学生弱，也不能简单理解成“少学了 filler”

因为这版学生即使做了 blank / non-blank KD，也没有把 repeated hard case 救回来。

所以主问题仍然更像：

- 困难样本上的 prefix/path 能力没建立起来

## 5. 这次实验的最终结论

这版可以判成失败，而且是比较明确的失败：

1. 原始 `test split` 没过
2. streaming 总量没有改善
3. repeated hard case 甚至比 `119.pt` 更差

因此当前不建议做的事是：

- 不继续做 KD-only sweep
- 不把 `output_kd_from119_v1` 当作新的训练起点

## 6. 对下一步的直接启发

这次实验真正给出的信息是：

- 单靠输出蒸馏，不足以解决当前 `199K` 学生的核心问题
- 主矛盾仍然在困难样本分布本身

因此下一步更合理的方向应该是：

1. 回到当前最好学生 `119.pt`
2. 先挖 `teacher-hit / student-miss` hard positives
3. 先做 hard positive replay
4. 如果 replay 有帮助，再考虑把 output KD 作为辅助项叠加回来

也就是说：

- 下一步主线应从 “KD-only” 切到 “hard positive replay first”
