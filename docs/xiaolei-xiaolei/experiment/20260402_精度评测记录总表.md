## 三、外部 test.list 固定阈值回灌统计

说明：

- 本表不重新在外部集上调阈值，而是直接沿用各模型在原始 `test split` 上选出来的阈值。
- 目的是看“如果部署时沿用 test split 调出来的阈值，外部集真实召回和误唤醒会怎样”。
- `误唤醒次数` 的统计口径是：一个 wav 只要出现至少一次误唤醒，就记为一次误唤醒，因此格式写成 `x/y`，其中 `y` 是负样本总数 `763`。
- 当前只统计已经实际跑过外部集的模型；`avg_30.pt` 如需补充，需要先补跑对应外部评测目录。

## 四、当前阶段结论

1. 在原始 test split 上，top20 weight surgery 已经不弱于 baseline2599。
2. `小雷小雷` 在原始 test split 上由 baseline `98.73%` 提升到 top20 surgery `99.25%`，同时总参数量从 `756,133` 降到 `392,494`。
3. `小雷快拍` 在原始 test split 上，baseline 与 top20 surgery 基本持平，都是 `99.87%` accuracy、`0.13%` FRR。
4. 在外部 test.list 上，按统一口径 `FA/h<=1.0` 比较，baseline2599 `79.pt` 为 `74.35%`，top20 surgery `159.pt` 提升到 `91.57%`。
5. 如果不重新在外部集调阈值，而是直接沿用原始 test split 上的阈值，则两版模型在外部集上都能保持约 `98%` 召回，但误唤醒明显偏高：baseline `54/763`，top20 surgery `50/763`。
6. 用户之前记住的“约 96% 召回”对应的是 baseline2599 在外部集更宽松的高误唤醒工作点，不是当前这份总表统一采用的 `1 FA/h` 工作点。
7. 后续蒸馏、小模型、S3 蒸馏结果应继续追加到上面三张总表中。
