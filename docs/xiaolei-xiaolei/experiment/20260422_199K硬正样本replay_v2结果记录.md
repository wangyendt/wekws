# 20260422 199K 硬正样本 replay v2 结果记录

## 1. 这组实验的 intuition

`hardpos_replay_from119_v1` 跑完以后，最合理的解释是：

- 首轮 replay 方向本身不一定错
- 但 replay 信号太弱
- 同时 `from119` 的 finetune 漂移太强

所以 `v2` 想验证的是一个更具体的问题：

- 如果把 replay 明显放大，同时缩短 schedule、降低学习率，能不能把 hard positive 干预真正做出效果

也就是把 `v1` 的假设具体化成：

- `replay_factor=3` 太低
- `finetune 60 epoch` 太长
- `1e-4` 太激进

## 2. 这次具体做了什么

### 2.1 数据配置

仍然沿用 `v1` 挖出来的同一批 hard positives：

- `826` 条 `teacher-hit / student-miss`

但把 replay 强度从：

- `replay_factor = 3`

提升到：

- `replay_factor = 10`

这意味着：

- 新增样本从 `2478` 条提高到 `8260` 条

### 2.2 训练配置

从 `119.pt` 起步，改成更保守的 finetune：

- `finetune_epochs = 30`
- `finetune_lr = 5e-5`
- `head_lr_ratio = 0.1`

实验目录：

- `exp/fsmn_ctc_xlxl_distill_199k_hardpos_replay_from119_v2`

## 3. 结果

### 3.1 原始 test split

`avg_30.pt` 统一口径结果：

- `小雷小雷 = 93.55%`
- `小雷快拍 = 99.09%`

对比：

- `v1 = 93.48% / 99.35%`
- `119.pt = 96.70% / 99.61%`

这组数字的含义很直接：

- `小雷小雷` 只比 `v1` 高了 `0.07`
- `小雷快拍` 反而比 `v1` 低了 `0.26`

所以 `v2` 没有带来实质性回升。

### 3.2 全量 streaming eval

全量 streaming 结果与静态完全一致：

- `小雷小雷 = 93.55%`
- `小雷快拍 = 99.09%`

说明：

- `v2` 也没有出现“静态一般，但流式更好”的隐藏收益

### 3.3 repeated hard case

两条代表样本：

- `0031 x3`
- `0021 x3`

在 `v2` 上的结果依然是：

- 学生只触发 `1/3`
- 且都只落在 `segment 1`

对比：

- 老师仍然是 `3/3`
- 旧学生 `119.pt` 至少还能做到 `2/3`

所以在部署更相关的 repeated hard case 上，`v2` 依然没有把问题救回来。

## 4. 这次结果代表什么

### 4.1 `v1` 的解释已经被直接验证

`v2` 其实已经把 `v1` 后最自然的解释测掉了：

- replay 不够强
- schedule 太长

因为这次同时做了：

- 更强 replay
- 更短 schedule
- 更小学习率

但结果仍然几乎没动。

### 4.2 replay-only 方向已经出现明显平台期

从 `v1 -> v2` 看：

- `93.48% -> 93.55%`

这个量级已经可以认为是“基本没变”。

更糟的是：

- `小雷快拍` 还退了
- hardcase 还是 `1/3`

所以当前更合理的判断是：

- replay-only 这条线已经明显平台期

### 4.3 主矛盾不只是训练分布覆盖

如果问题主要只是：

- hard positives 不够多

那么 `v2` 这种更强 replay 至少应当在 repeated hard case 上出现一点恢复。

现在没有。

这更支持：

- 问题不只是“数据没喂够”
- 还需要更显式地改变 loss / sample weighting，让这批样本在训练里真正更重

## 5. 当前结论

`hardpos_replay_from119_v2` 可以直接判成：

- 对 `v1` 没有实质性改善
- 仍明显低于 `119.pt`
- repeated hard case 仍然是 `1/3`

所以这版实验的价值在于：

- 它帮我们确认 replay-only 不是当前主线答案

而不是：

- 它本身是一个可保留基线
