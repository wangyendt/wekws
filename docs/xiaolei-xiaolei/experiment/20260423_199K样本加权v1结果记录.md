# 20260423 199K 样本加权 v1 结果记录

## 1. 实验 intuition

在 `replay-only v2` 之后，最自然的下一步是把那 `826` 条：

- `teacher-hit / student-miss`

的 hard positives 从“复制 rows”改成“loss 层显式加权”。

首轮 `v1` 采用的是：

- `hard_weight = 5.0`
- 普通样本权重 `1.0`
- 对 `CTC / MSE / KD` 统一做 **all-loss weighting**

它要验证的问题是：

- 如果不改变 train list 本身，只直接放大 hard positive 的训练权重，能不能比 replay-only 更有效地改善 `小雷小雷` 召回与 repeated hard case。

## 2. 实验设置

### 2.1 数据来源

- 原始训练集：`data_xlxl_0327_ctc_v1_clean`
- hard positives：`826` 条
- 权重映射文件：`exp/xlxl_aux/hardpos_weight_from119_v1.json`

### 2.2 训练设置

- 起点：`exp/fsmn_ctc_xlxl_distill_199k/119.pt`
- `align_epochs = 120`
- `finetune_epochs = 30`
- `finetune_lr = 5e-5`
- `head_lr_ratio = 0.1`
- `finetune_mse_weight_start = 0.3`
- `finetune_mse_weight_end = 0.1`
- 不叠 `output KD`
- sample weighting scope：`all`

实验目录：

- `exp/fsmn_ctc_xlxl_distill_199k_hardpos_weight_from119_v1`

## 3. 结果

### 3.1 原始 test split

- `小雷小雷 = 90.85%`
- `小雷快拍 = 98.43%`

### 3.2 全量 streaming

与静态完全一致：

- `小雷小雷 = 90.85%`
- `小雷快拍 = 98.43%`

### 3.3 repeated hard case

#### `0031 x3`

- 学生：`1/3`
- 只触发 `segment 1`

#### `0021 x3`

- 学生：`1/3`
- 只触发 `segment 1`

老师两条都仍然是：

- `3/3`

## 4. 对比前序实验

相对 `hardpos_replay_from119_v2`：

- `小雷小雷: 93.55% -> 90.85%`
- `小雷快拍: 99.09% -> 98.43%`

相对当前最好学生 `119.pt`：

- `小雷小雷: 96.70% -> 90.85%`
- `小雷快拍: 99.61% -> 98.43%`

## 5. 结论

这轮可以直接判失败，而且不是“小退一点”，而是明显带坏了模型。

结论不是：

- 样本加权这条线无效

而是：

- **当前这版 all-loss weighting 明显方向不对**

更具体地说，最像的问题是：

- hard positive 权重不仅放大了 CTC 分支，也一起放大了 `MSE` 分支
- 结果学生被拉离了原本还能工作的解
- 但 repeated hard case 的多段 prefix/path 稳定性并没有被修好

## 6. 下一步

下一步不建议继续做：

- `hard_weight = 8/10`

也不建议继续沿用：

- `all-loss weighting`

更合理的下一步是：

- 保留同一批 `826` 条 hard positives
- 保留 `hard_weight = 5.0`
- 先把 sample weighting 范围收窄到：
  - `CTC only`

先验证真正的问题是不是：

- 把 hard weight 施加到 `MSE/KD` 分支后把模型带偏了
