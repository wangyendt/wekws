# 20260423 199K 样本加权 ctc_only v1 结果记录

## 1. 实验动机

上一版 `hardpos_weight_from119_v1` 把 `826` 条 hard positive 的权重同时施加到 `CTC / MSE / KD / blank-KD`，结果明显变差。

这版想验证的问题更具体：

- 失败是不是因为把 `MSE/KD` 也一起放大了。
- 如果只对 `CTC` 做样本加权，能不能更直接地拉回关键词路径，而不破坏学生 backbone 已有表征。

## 2. 实验配置

- 起点模型：`exp/fsmn_ctc_xlxl_distill_199k/119.pt`
- hard positive 来源：`teacher-hit / student-miss`
- hard positive 数量：`826`
- 权重设置：`hard_weight=5.0`
- 加权范围：`CTC only`
- 实验目录：`exp/fsmn_ctc_xlxl_distill_199k_hardpos_weight_ctc_only_from119_v1`

## 3. 结果

### 3.1 原始 test split

`avg_30.pt`

- `小雷小雷 = 91.00%`
- `小雷快拍 = 98.43%`

对比：

- 比 `119.pt` 明显差：`96.70% / 99.61% -> 91.00% / 98.43%`
- 只比 all-loss weighting v1 略好：`90.85% / 98.43% -> 91.00% / 98.43%`

### 3.2 早期 checkpoint

- `124.pt = 93.03% / 98.82%`
- `129.pt = 90.40% / 98.30%`

这说明：

- 早期确实比 `avg_30` 好一些
- 但 `124.pt` 也仍明显低于 `119.pt`
- 所以问题不只是“后期漂移”，而是这条加权方向本身就没有把模型拉回正确区域

### 3.3 全量 streaming

`test_infer_stream_avg30_chunk300`

- `小雷小雷 = 91.00%`
- `小雷快拍 = 98.43%`

和静态完全一致，没有隐藏的 streaming 收益。

### 3.4 repeated hard case

两条代表性 hard case：

- `0031 x3`：学生 `1/3`，只触发 `segment 1`
- `0021 x3`：学生 `1/3`，只触发 `segment 1`

老师仍是稳定 `3/3`。

## 4. 结论

这版 `CTC only weighting` 可以判失败。

原因不是单纯 `avg_30` 混入了坏 checkpoint，而是：

- 主 checkpoint `avg_30` 明显低于 `119.pt`
- 早期 checkpoint `124.pt` 也仍明显低于 `119.pt`
- streaming 没改善
- repeated hard case 仍是 `1/3`

因此样本加权这条线到目前为止不值得继续堆 `hard_weight` 或继续做 `v2`。

下一步应切到更克制的方向：只动输出头，不再改整个 backbone。
