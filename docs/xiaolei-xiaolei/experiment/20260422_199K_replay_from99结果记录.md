## 1. 背景

这次 `replay_from99` 是原计划里 `79 / 99 / 119` continuation seed sweep 的最后一条。

目的只有一个：

- 验证 `99.pt` 这个中间 checkpoint 继续训，是否会比当前最好学生 `119.pt` 更好

如果不能超过 `119.pt`，则当前 `FA + CTC continuation` 路线阶段性收口，不再继续沿这条线扩新分支。

## 4. 当前结论

这条分支可以直接判失败，理由很明确：

1. `小雷小雷` 从 `119.pt` 的 `96.70%` 掉到 `91.53%`
2. `小雷快拍` 从 `99.61%` 掉到 `96.99%`
3. `threshold=0.062` 仍只有 `0.34 FA/h`，说明问题不只是阈值，而是模型本身召回已经塌了

因此：

- 不补跑 `replay_from99` 的外部 `test.list`
- 不再继续扩新的 continuation seed 或 from119 变体
- 下一步转向针对漏检模式的训练干预

## 5. 下一步衔接

当前后续主线应切到：

1. 以 `119.pt` 作为当前最好学生基线
2. 挖 `teacher-hit / student-miss` 的 hard positive
3. 按“前缀起步失败 / 重复扩展失败 / 老师也不稳定”三类分桶
4. 优先做 hard positive replay，再看是否需要样本加权或 prefix/path 约束
