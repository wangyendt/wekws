## 4. 结果的含义

### 4.1 这版没有通过主 gate

这次的主 gate 是：

- 原始 `test split` 上至少超过 `119.pt`

这条没有满足。

因为：

- `小雷小雷`：`96.70% -> 92.35%`
- `小雷快拍`：`99.61% -> 99.22%`

所以就原始 `test split` 而言，这版不能算成功。

### 4.2 这不是“略差一点”，而是 output KD 单独还不够

它比 `replay_from99`、`replay_from119` 和 `ft_from119_ctc_v1` 都更好一些，
说明“贴近老师输出行为”这个方向本身未必错。

但它仍然明显输给：

- `119.pt`
- `replay_from79`

所以当前更准确的判断是：

- output-level KD 单独做，还不足以把召回拉回当前最好学生

### 4.3 `threshold=0.000` 且 `fa_per_hour=0.00` 的含义

这次两条关键词都选到了：

- `threshold = 0.000`

并且 `小雷小雷` 在这个点仍然只有：

- `accuracy = 92.35%`
- `fa_per_hour = 0.00`

这说明问题不像是“阈值调得不对”，更像：

- 这版模型整体变得更保守
- 输出分布虽然可能更平滑，但 recall 没被救回来

所以这次的失败更像是：

- output 行为蒸馏方向对，但力度还不够，或者缺少困难样本干预

### 4.4 还不能只靠这个结果否定 output KD

虽然原始 `test split` gate 没过，但这版训练的目标并不只看静态 accuracy，
还包括：

- streaming 行为
- repeated hard case 的第一段起步

所以这次结果更准确的解读应该是：

- output KD alone 没能直接把静态精度拉过 `119.pt`
- 但仍值得继续看 streaming / repeated hard case 是否有定向改善

## 5. 当前收口

这版实验可以先记成：

- 原始 `test split` 上失败
- 但提供了一个很清楚的信息：
  - 只加 output-level KL + blank/non-blank KD，还不够

因此下一步不该继续做更多 continuation，而应优先判断：

1. 这版在 streaming / repeated hard case 上有没有比 `119.pt` 更接近老师
2. 如果没有，则下一版应转向：
   - hard positive replay
   - teacher-hit / student-miss 样本加权
   - 必要时再叠加 output KD，而不是单独依赖它
