## 结论

1. 在外部 `test.list` 上，这版 baseline 的泛化明显弱于原始 test split。
2. 按 `FA/h<=1.0` 的统一口径，`小雷小雷` 只能做到 `accuracy 74.35% / FRR 25.65% / FA 0.99/h`。
3. 当前评测使用的 baseline 2599 模型参数量是 `756,133`；因此这里的退化不是“模型尺寸变化”导致的，而是数据域变化后的泛化问题。
4. 如果直接看当前 `score.txt` 的默认判决，则召回较高（`98.45%`），但误唤醒也明显偏高（`57 / 763` 负样本被误触发）。
5. 外部负样本未发现明显错标，因此下一步优化重点应放在模型压缩/蒸馏/阈值策略，而不是先怀疑这份外部 test 的负样本标签质量。
