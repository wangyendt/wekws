# 20260422 重复 hard case 流式稳定性实验记录

## 1. 这次实验回答什么问题

前面的单条 `logit / prefix` 图说明：

- 老师经常在前面给出一个 `<filler>` 过渡
- 学生更常见的是前半段一直被 `<blk>` 压住

但单条短音频还不能回答一个更接近部署的问题：

- 如果把同一条 hard case 连续拼接成更长音频，老师和学生在流式模式下是否还能稳定多次触发
- 老师是不是会在每一段关键词前都重新给出一次 `<filler>`

因此这次做了“重复 hard case 的流式稳定性实验”。

## 3. 结果

### 3.1 流式触发次数

老师：

- 共触发 `3` 次
- 分别落在 `segment 0 / 1 / 2`
- 三次都命中目标关键词 `小雷小雷`

学生：

- 共触发 `2` 次
- 只落在 `segment 1 / 2`
- 第一段 `segment 0` 漏掉

这说明：

- 老师在长音频串联场景下具备稳定的多次流式触发能力
- 学生在更接近部署的长序列流式场景下，前段更容易漏掉

### 3.2 `<filler>` 是否每段都会出现

结论是否定的。

老师虽然在整段非流式 greedy 里有一个 `<filler>`：

- `"<filler> 小 雷 小 雷 小 雷 小 雷 小 雷 小 雷"`

但从分段峰值看：

- 第 0 段 `<filler>` 峰值约 `0.6108`
- 第 1 段 `<filler>` 峰值约 `2.16e-6`
- 第 2 段 `<filler>` 峰值约 `4.24e-6`

也就是说：

- 老师不会在每一段拼接出来的关键词前都重新给出一个强 `<filler>`
- `<filler>` 更像整个 utterance 开头的前导过渡状态，而不是每个关键词实例的硬模板

学生在三段里都几乎没有有意义的 `<filler>` 响应：

- 第 0 段约 `5.35e-3`
- 后两段也只有 `1e-6` 量级

### 3.3 非流式整段路径

老师非流式 greedy：

- `<filler> 小 雷 小 雷 小 雷 小 雷 小 雷 小 雷`

学生非流式 greedy：

- `雷 小 雷 小 雷 雷 小 雷`

这说明在长音频里：

- 老师仍能保持完整重复结构
- 学生更容易把多段内容串成不稳定路径，尤其前面一段不完整

## 4. 当前结论

这次实验最重要的结论有三条：

1. 老师的优势不是“每段前面都必须出现一个 `<filler>`”
2. 老师真正更强的地方是：在长音频流式场景下仍能稳定保住多次关键词触发
3. 学生当前更像“多次流式触发能力不足”，而不只是“没有学会 `<filler>`”

因此后续训练目标应更准确地写成：

- 让学生更接近老师的多次流式触发能力
- 强化前缀起步与重复结构保持
- 不把“每段都出现 `<filler>`”当成必须硬逼近的目标
